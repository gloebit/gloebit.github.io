The Gloebit Sandbox

Gloebit provides a sandbox server that helps you start integrating Gloebit services quickly. It also provides all Gloebit functionality so you can fully test your design before switching over to the production Gloebit server.

This sandbox walk-through and guide takes you through the steps to get your design up and and going in the Gloebit sandbox. Let's get started.

Gloebit Merchant Prequisites

Whether on the production or sandbox server, a Gloebit merchant needs the following to use Gloebits:

  1. A merchant Gloebit account (on the server)
  2. An integrated Gloebit API

This guide walks you through both requirements in that order. However, if you would rather immediately start integrating an API, you can skip this step and intstead use the test merchant account.

Users and merchants create accounts on Gloebit's website. An application associated with a merchant account uses OAuth2 to get a user's permission to interacts with their account.

Users purchase gloebits on Gloebit's website -- Gloebit takes care of interfacing with various payment providers. Once a user has a balance of gloebits, an application can exchange entitlements or digital inventory for the user's gloebits.
