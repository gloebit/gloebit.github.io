Flow for OpenSim user authorizing an application.

Dependent upon how the Gloebit Money Module is configured and the corresponding application is created on Gloebit, this application could represent an entire grid, a set of regions, or a single region on a grid.

Authorization Request from OpenSim viewer

Authorization form on Gloebit

The user is delivered a form on the Gloebit website from which they can approve or decline authorization of this application corresponding to a grid or set of regions. If the user is not logged in, they would land at the Gloebit login page where they can log in or create an account, after which they would be directed to this form.

Authorization response

After authorizing, the user is redirected to a url supplied by the application. The default from the module displays this simple message.
